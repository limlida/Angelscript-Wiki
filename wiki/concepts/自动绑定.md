## 概述
Angelscript 插件在引擎启动时自动遍历 Unreal 的反射数据，将 C++ 类型、属性和函数自动绑定到脚本中。

## 核心原则

> **如果可以从 Blueprint 使用，就应该可以从 Angelscript 使用。**

## 绑定规则

### 类绑定
| C++ 标记 | 自动绑定条件 |
|---|---|
| `UCLASS()` + `BlueprintType` | ✅ 自动绑定 |
| 包含 `BlueprintCallable` 函数 | ✅ 自动绑定 |
| `NotInAngelscript` 元数据 | ❌ 跳过 |

### 结构体绑定
| C++ 标记 | 自动绑定条件 |
|---|---|
| `USTRUCT()` + `BlueprintType` | ✅ 自动绑定 |
| 包含 Blueprint 可访问属性 | ✅ 自动绑定 |
| `NoAutoAngelscriptBind` 元数据 | ❌ 跳过 |

### 属性绑定
| C++ 标记 | 脚本中行为 |
|---|---|
| `BlueprintReadWrite` | 可读写 |
| `BlueprintReadOnly` | 只读（`const`） |
| `ScriptReadWrite` | 仅脚本可读写，不暴露给 Blueprint |
| `ScriptReadOnly` | 仅脚本只读 |
| `EditAnywhere` 等 | 仅在 `default` 语句中可访问 |
| `NotInAngelscript` | 跳过 |

### 函数绑定
| C++ 标记 | 脚本中行为 |
|---|---|
| `BlueprintCallable` / `BlueprintPure` | 自动绑定 |
| `ScriptCallable` | 仅脚本可用 |
| `BlueprintImplementableEvent` | 可从脚本重写 |
| `BlueprintNativeEvent` | 可从脚本重写 |
| `NotInAngelscript` | 跳过 |
| 废弃标记 | 不绑定 |

### 静态函数
- 绑定为命名空间全局函数
- 类名经过命名空间简化（详见 [[Source_FunctionLibraries]]）

### 枚举绑定
- 所有 `UENUM()` 自动可用
