## 概述
Angelscript 提供了完整的测试框架，包括 xUnit 风格的单元测试和可播放游戏场景的集成测试。

## 单元测试

### 声明
```angelscript
void Test_NameOfTheTestCase(FUnitTest& T)
{
    T.AssertTrue(Condition);
    T.AssertEquals(Expected, Actual);
    T.AssertNotNull(ObjectRef);
}
```

### 运行方式
- **热重载自动运行**：保存文件后在 Output Log 中查看结果
- **Session Frontend**：在 Automation Tab 的 "Angelscript" 分类下
- **命令行**：`-run=AngelscriptTest`

### 命名约定
- 测试文件：`File_Test.as`（对应 `File.as`）

## 集成测试

### 声明
```angelscript
void IntegrationTest_MyTestName(FIntegrationTest& T)
{
    T.AddLatentAutomationCommand(UMyLatentCommand());
}
```

### 测试地图
- 默认需要 `/Content/Testing/IntegrationTest_MyTestName.umap`
- 可自定义：`FString IntegrationTest_MyTestName_GetMapName()`

### 潜伏自动化命令
用于需要跨多帧执行的测试：
```angelscript
class UMyLatentCommand : ULatentAutomationCommand
{
    UFUNCTION(BlueprintOverride)
    bool Update()
    {
        return TestConditionMet;
    }

    UFUNCTION(BlueprintOverride)
    FString Describe() const
    {
        return "Description of current test state";
    }
}
```

- `Update()` 返回 `true` 表示测试通过
- 默认超时 **5 秒**
- `default bAllowTimeout = true` 允许超时（用于测试某事*不会*发生）

### 动态测试用例
```angelscript
void ComplexIntegrationTest_Name_GetTests(TArray<FString>& OutTestCommands)
{
    for (auto Item : GetAllItems())
    {
        OutTestCommands.Add(Item.GetName());
    }
}
```

### 客户端/服务器模式
- 默认以客户端/专用服务器模式运行
- 可在项目设置中切换为独立模式

## 代码覆盖率

### 启用方式
- 项目设置 > Editor > Angelscript Test settings
- 或命令行参数 `-as-enable-code-coverage`

### 报告
- 输出到 `Saved/CodeCoverage/index.html`
- 可查看每个文件的行覆盖率

### 注意
- 启用代码覆盖率会使编辑器启动慢约 20 秒
