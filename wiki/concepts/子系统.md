## 概述
子系统（Subsystem）是 Unreal 的单例模式实现，将常用功能组织为易于访问的全局对象。Angelscript 支持使用和创建子系统。

## 使用子系统

### 获取子系统实例
```angelscript
auto Subsystem = ULevelEditorSubsystem::Get();
Subsystem.NewLevel("/Game/NewLevel");
```

### 注意事项
- 许多子系统是**编辑器子系统**，不能在打包游戏中使用
- 编辑器子系统应在 `#if EDITOR` 块中使用，详见 [[编辑器脚本]]

## 创建脚本子系统

### 可用基类
| 基类 | 作用域 | 生命周期 |
|---|---|---|
| `UScriptEngineSubsystem` | 引擎全局 | 引擎启动到关闭 |
| `UScriptGameInstanceSubsystem` | 游戏实例 | 游戏实例存在期间 |
| `UScriptWorldSubsystem` | 世界/关卡 | 关卡加载到卸载 |
| `UScriptLocalPlayerSubsystem` | 本地玩家 | 玩家连接到断开 |
| `UScriptEditorSubsystem` | 编辑器 | 编辑器运行期间 |

### 示例：世界子系统
```angelscript
class UMyGameWorldSubsystem : UScriptWorldSubsystem
{
    UFUNCTION(BlueprintOverride)
    void Initialize()
    {
        Print("MyGame World Subsystem Initialized!");
    }

    UFUNCTION(BlueprintOverride)
    void Tick(float DeltaTime)
    {
        // 每帧逻辑
    }

    UFUNCTION()
    void LookAtMyActor(AActor Actor)
    {
    }
}
```

### 使用自定义子系统
```angelscript
auto MySubsystem = UMyGameWorldSubsystem::Get();
MySubsystem.LookAtMyActor(nullptr);
```

### 本地玩家子系统
获取时需传入 `ULocalPlayer` 或 `APlayerController`：
```angelscript
ULocalPlayer RelevantPlayer = Gameplay::GetPlayerController(0).LocalPlayer;
auto MySubsystem = UMyPlayerSubsystem::Get(RelevantPlayer);
```

### Blueprint 交互
- `UFUNCTION()` 声明的方法自动可在 Blueprint 中访问
