## 概述
预编译脚本功能可加速打包版本的启动时间，并通过 C++ 转译实现接近原生的运行时性能。

## 工作原理

### 预编译缓存
- 正常启动时，插件需加载、解析和编译所有 `.as` 文件
- 预编译缓存直接加载编译后的字节码，跳过上述步骤
- 显著提升启动速度，特别是在 CPU 和硬盘较慢的平台上

### C++ 转译（JIT）
- 生成等效 C++ 代码文件
- 重新编译后利用 JIT 系统用原生 C++ 替代虚拟机字节码执行
- **显著提升运行时性能**

## 使用流程

### 1. 生成预编译数据
```
ProjectName.exe -as-generate-precompiled-data
```
生成 `ProjectName/Script/PrecompiledScript.Cache`

### 2. 生成 C++ 转译代码
自动生成 `AS_JITTED_CODE/` 文件夹，包含每个脚本文件对应的 C++ 文件。

### 3. 编译 C++ 转译代码
- 将 `AS_JITTED_CODE/` 复制到项目源码目录
- 在 Build.cs 中添加 `AngelscriptCode` 依赖
- 重新编译游戏

## 兼容性规则

| 操作 | 需要重新生成缓存？ | 需要重新编译 C++？ |
|---|---|---|
| 修改 `.as` 脚本 | ✅ | ❌ |
| 重新构建游戏 `.exe` | ✅ | ✅ |
| 添加 C++ 转译代码 | ❌ | ✅（仅一次） |

**关键规则**：预编译缓存必须由**相同的二进制文件**生成和加载。

## 限制
- 使用预编译缓存时**热重载被禁用**
- 可用 `-as-development-mode` 绕过缓存加载
