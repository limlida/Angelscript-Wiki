## 概述
Angelscript 中的结构体是值类型，与类（UObject 引用类型）有本质区别。理解结构体的声明、传递和引用机制对编写高效脚本至关重要。

## 声明结构体

```angelscript
struct FExampleStruct
{
    UPROPERTY()
    float ExampleNumber = 4.0;

    UPROPERTY()
    FString ExampleString = "Example String";

    float HiddenNumber = 3.0;
};
```

### 关键限制
- **结构体不能有 `UFUNCTION()`**
- 可以有普通脚本方法（但不可从 Blueprint 调用）

## 参数传递

### 默认行为：只读
```angelscript
void UseStruct(FExampleStruct Struct)
{
    // Struct 是只读的，不能修改
}
```

### 可修改引用：`&`
```angelscript
void ModifyStruct(FExampleStruct& Struct)
{
    Struct.ExampleNumber = 42.0;
}
```

### 输出参数：`&out`
```angelscript
UFUNCTION()
void OutputStruct(FExampleStruct&out OutputStruct, bool&out bSuccess)
{
    FExampleStruct Result;
    Result.ExampleNumber = 42.0;
    OutputStruct = Result;
    bSuccess = true;
}
```
- `&out` 在 Blueprint 中创建**输出引脚**
- 适用于结构体和基本类型

## 自动引用优化

**重要**：脚本函数的结构体参数始终以 `const &` 传递。

```angelscript
// 以下两种声明完全等价
void Func1(FVector Vec) { }
void Func2(const FVector& Vec) { }
```

- 无需手动添加 `const &` 优化性能
- 这是为了避免要求脚本编写者理解引用语义

## 结构体 vs 类

| 特性 | 结构体 (struct) | 类 (class) |
|---|---|---|
| 类型 | 值类型 | 引用类型 |
| 基类 | 无 | UObject |
| UFUNCTION | ❌ 不支持 | ✅ 支持 |
| GC 管理 | 不需要 | 自动 |
| 传递方式 | const &（自动） | 引用 |
