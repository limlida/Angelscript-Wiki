## 概述
Angelscript 支持编辑器专用代码，允许在编辑器中使用仅编辑器可用的 API（如 Actor 标签、编辑器子系统等），同时确保打包版本不会编译失败。

## `#if EDITOR` 预处理器

### 基本用法
```angelscript
class AExampleActor : AActor
{
    #if EDITOR
    default PivotOffset = FVector(0, 0, 10);
    #endif

    UFUNCTION(BlueprintOverride)
    void ConstructionScript()
    {
        #if EDITOR
        SetActorLabel("Example Actor Label");
        #endif
    }
}
```

### 其他预处理器宏
| 宏 | 含义 |
|---|---|
| `EDITOR` | 当前是否在编辑器中运行 |
| `EDITORONLY_DATA` | 编辑器专用属性是否可读 |
| `RELEASE` | 是否为 Shipping 或 Test 构建配置 |
| `TEST` | 是否为 Debug、Development 或 Test 构建配置 |

## 编辑器专用目录

以下目录名在打包构建中被完全忽略：
- **`Editor/`** - 编辑器专用脚本
- **`Examples/`** - 示例脚本
- **`Dev/`** - 开发辅助脚本

## 模拟打包模式

使用 `-as-simulate-cooked` 命令行参数可模拟打包环境，检测编辑器专用代码是否会编译失败。

### 命令行测试
```
UnrealEditor-Cmd.exe "MyProject.uproject" -as-simulate-cooked -run=AngelscriptTest
```

### 工作原理
- 编辑器专用属性和类不可用
- `#if EDITOR` 块被编译排除
- 适用于 CI 流水线中的自动化验证

## 最佳实践
1. 编辑器专用代码始终包裹在 `#if EDITOR` 中
2. 将编辑器专用类放在 `Editor/` 目录下
3. 打包前使用模拟打包模式验证脚本编译
