### UNiagaraDataChannelWriteModuleData


Helper class to display properties in the write data channel module wizard

**属性**:

- `uint AllocationCount [How many elements to allocate for writing per frame? Usage depends on AllocationMode.]`
- `ENiagaraDataChannelAllocationMode AllocationMode [How should we allocate the buffer into which we write data.]`
- `UNiagaraDataChannelAsset DataChannel [The source asset to read from]`
- `ENiagaraDataChanneWriteModuleMode WriteMode [Should the generated module append to the data channel or write to an existing element?]`
- `bool bAutoTransformPositionData [If true then position inputs are automatically transformed from simulation space to world space, so the write works correctly for localspace emitters.]`
- `bool bPublishToCPU [Whether the data generated by the niagara data interface should be published to CPU emitters in other Niagara systems. Setting this to true will have a minor performance impact.]`
- `bool bPublishToGPU [Whether the data generated by the niagara data interface should be published to GPU emitters in other Niagara systems. Setting this to true will have a minor performance impact.]`
- `bool bPublishToGame [Whether the data generated by the niagara data interface should be published to the world game data channel. This is required to allow game BP and C++ to read this data. Setting this to true will have a minor performance impact.]`
- `bool bUpdateDestinationDataEveryTick [Whether this DI should request updated destination data from the Data Channel each tick.
Depending on the data channel this could be an expensive search so we should avoid doing this every tick if possible.]`
